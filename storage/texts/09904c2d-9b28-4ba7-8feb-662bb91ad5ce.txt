Abstract
ChatSphere is a modern real-time chat application designed to provide an organized
andefficientcommunicationplatform. ThemainobjectiveofChatSphereistoseparate
professional and casual conversations by introducing two distinct modes: Work Mode
and Private Mode. This separation helps users manage important discussions and
personal chats more effectively.
Theapplicationbeginswithalandingpagethatintroducestheprojectandprovides
optionsforuserregistrationandlogin. Duringregistration, usersenteressentialdetails
such as name, email, password, and a security question-answer pair, which supports
the password recovery process. If a user forgets their password, the system verifies
identity by asking the stored security question, and only upon correct answer the
password can be reset.
After successful login, users are directed to a homepage where they can select
either Work Mode or Private Mode. Based on the selected mode, users access a
chat interface that supports direct messaging, room-based group chats, and advanced
features such as starred messages, message categorization, file storage, and real-time
message editing. The application also provides message search functionality using
stringsearchingmethods. Additionally,ChatSpheresupportsthemeswitchingbetween
dark and light mode for better user experience.
Overall, ChatSphere aims to provide a secure, scalable, and user-friendly commu-
nication system with enhanced organization features like message categorization and
file management, making it suitable for both personal and professional use cases.

Chapter 1
Introduction
1.1 Project Overview
ChatSphere is a real-time chat web application that allows users to communicate
through direct messaging and group rooms. The application introduces two sepa-
rate chat environments: Work Mode and Private Mode. This enables users to keep
professional conversations and personal conversations separate within the same plat-
form. ChatSphere also includes organizational features such as starred messages with
category grouping and file storage with categorization.
1.2 Purpose of the Project
The purpose of ChatSphere is to provide a messaging platform that is structured and
organized. Many chat applications focus mainly on communication but do not offer
efficient ways to manage important messages or store files in a categorized manner.
ChatSphere solves this by introducing message starring, category management, and a
dedicated file storage system.
1.3 Scope of the Project
The scope of ChatSphere includes:
• User authentication system (register, login, logout).
• Forgot password feature using security question verification.
• Mode-based chatting (Work Mode and Private Mode).
• One-to-one direct chat with registered users.
ChatSphere Project Report 2
• Room-based group chatting with join/leave functionality.
• Starred message feature with user-defined categories.
• File storage management with categories.
• Real-time messaging and message editing.
• Message search feature.
• User profile settings for updating name and password.
• Theme switching between light and dark mode.
1.4 Problems in Existing Systems
Existing messaging applications face several limitations such as:
• No proper separation between professional and personal chats.
• Difficulty in managing important messages due to lack of categorization.
• No structured storage system for files shared within chats.
• Limited search functionality.
• Poor customization options for user interface themes.
1.5 Objectives of the Project
The main objectives of ChatSphere are:
• To provide a secure and user-friendly chat application.
• To enable users to separate professional and casual chats.
• To allow users to organize important messages using starred categories.
• To provide a categorized file storage feature.
• To support real-time communication and message editing.
• To provide an efficient message searching feature.

storage.

3.3 Data Flow Explanation
The data flow begins with user authentication. After login, the user selects a mode.
The system loads the corresponding chat environment. Messages are sent from the
frontend to backend and stored in database. For rooms, messages are broadcasted to
all members. Starred messages and file uploads are stored separately with category
mapping.
3.4 Database Design
The database includes tables such as:
• Users Table (name, email, password, security question, answer)
• Messages Table (sender, receiver/room, content, timestamp, mode)
• Rooms Table (room name, description, members)
• Starred Messages Table (message id, user id, category id)
• Categories Table (category name, id , userId)
• Files Table (file name, file path, uploaded by, category id)

4.4 Forgot Password Module
The forgot password feature works as follows:
1. User clicks on Forgot Password.

This improves security and ensures only the legitimate user can reset their password.
